# Security Policy

## Reporting Vulnerabilities

If you discover a security vulnerability in Excel Chatbot AI, please email:
**security@nurasyid.com**

Do not open public GitHub issues for security vulnerabilities.

## Security Best Practices

### For Users
1. Keep the app updated to the latest version
2. Use strong passwords for any connected accounts
3. Be cautious with file permissions
4. Review watermarked files before sharing

### For Developers
1. Never commit `.env` files with sensitive data
2. Use environment variables for API keys
3. Validate all user inputs
4. Keep dependencies updated
5. Use HTTPS for all API communications

## Data Protection

- All uploaded images are processed locally
- Excel files are watermarked for tracking
- Analytics data is anonymized
- No personal data is stored

## Third-Party Libraries

We use the following libraries with security considerations:
- **React** - Regularly updated
- **ExcelJS** - Maintained and secure
- **Tesseract.js** - Open source OCR
- **Electron** - Latest security patches

## Compliance

- GDPR compliant
- No third-party tracking
- Local processing where possible
- Transparent data practices

## Updates

Security updates are released as patch versions (x.x.Z).
Update immediately when security patches are available.

## Acknowledgments

We appreciate the security research community's efforts in helping us maintain a secure application.
